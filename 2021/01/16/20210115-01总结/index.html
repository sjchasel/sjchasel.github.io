一些参数解释：

  • param_init_glorot & param_init 0: correct initialization of parameters;
    关于glorot的解释
    glorot_normal:Glorot正态分布初始化方法，也称作Xavier正态分布初始化，参数由0均值，标准差为sqrt(2 / (fan_in + fan_out))的正态分布产生，其中fan_in和fan_out是权重张量的扇入扇出（即输入和输出单元数目）
    glorot_uniform：Glorot均匀分布初始化方法，又成Xavier均匀初始化，参数从[-limit, limit]的均匀分布产生，其中limit为sqrt(6 / (fan_in + fan_out))。fan_in为权值张量的输入单元数，fan_out是权重张量的输出单元数。
    我猜第二个是全0初始化？

  • position_encoding: add sinusoidal position encoding to each embedding;
    正弦位置编码
    有关位置编码的详细解释

  • optim adam, decay_method noam, warmup_steps 8000: use special learning rate;
    用于设置学习率。第一个应该就是用adam方法，第二个应该是指使用noam这种衰减方式
    noam：
    主要和四个参数相关，当前步数(S), 预热步数($W_S$)， 网络层大小($L_S$)，原始学习速率($O_S$)

    详细解释可以再看:碎碎念：Transformer的细枝末节
    关于warmup:Warmup预热学习率

  • batch_type tokens, normalization tokens: batch and normalize based on number of tokens and not sentences;
    看不太懂

  • accum_count 4: compute gradients based on four batches;
    每四批计算一次梯度？

  • label_smoothing 0.1: use label smoothing loss.
    标签平滑是是分类问题中错误标注的一种解决方法。
    标签平滑Label Smoothing

为什么需要field？
torchtext 认为一个样本是由多个字段（文本字段，标签字段）组成，不同的字段可能会有不同的处理方式，所以才会有 Field 抽象。

  • Field: 定义对应字段的处理操作
  • Vocab: 定义了词汇表
  • Vectors: 用来保存预训练好的 word vectors

参考：pytorch学习笔记（十九）：torchtext

创建模型和优化器

padding是什么
padding（填充），是让表示句子的向量有相同长度的操作。很多模型都要求输出的长度相同，但是我们的句子有的长有的短。因此我们可以设定一个固定的长度/默认情况下使用语料库中最长句子的长度。当句子比这个长度短时，我们填充它（可以用0，应该也有其他方法），当句子长于这个长度时，默认情况下把开头多出来的一段截掉。这是对文本进行的预处理，处理完之后就可以进行embedding。

模型是RNN 编码器与解码器 注意力机制 编码器为双向
