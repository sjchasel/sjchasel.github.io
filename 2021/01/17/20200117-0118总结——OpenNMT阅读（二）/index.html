20210117——阅读OpenNMT官方文档(二)
2021-01-17|

之前的Library，我猜应该是介绍了一下onmt的大概结构？之后的应用都是通过修改yaml配置文件来进行的了。比如Translation、Summarization。还介绍了OpenNMT中引入的门控图神经网络（Gated Graph Neural Network，ggnn）的相关配置。这部分中关于Graph data format的没咋看懂。

接下来进入SCRIPT部分，里面介绍了整个流程中参数的细节。我只能当一个没有感情的翻译机器。

Build Vocab

build_vocab.py
是哪里的文件？？在源码包里没有找到。下面是它的用法。看起来应该是代指有build_vocab功能的py文件就行？

Configuration

  • -config, --config
    指yaml配置文件的路径。

  • -save_config, --save_config
    保存配置的路径

Data

  • -data, --data
    数据集和数据集的规范？

  • -skip_empty_level, --skip_empty_level
    当遇到空的example时

  1. silent：无视/跳过
  2. warning（默认）：无视/跳过 且发出警告
  3. error：抛出错误/停止运行
  • -transforms, --transforms
  1. filtertoolong：大概是清洗掉太长的数据？
  2. prefix：这个单词的意思是前缀，具体怎么操作我就不知道了
  3. switchout：SwitchOut: an Efficient Data Augmentation Algorithm for Neural Machine Translation 竟然搜出了一篇论文，看起来是数据增强的操作
  4. tokendrop：把标记去掉？
  5. tokenmask：把标记遮盖掉？
  6. sentencepiece：分句？
  7. bpe：是一种数据压缩算法，用来获得subword。NLP三大Subword模型详解：BPE、WordPiece、ULM
  8. onmt_tokenize：自己的标记方法？
  9. bart：BART: Denoising Sequence-to-Sequence Pre-training for Natural Language Generation, Translation, and Comprehension一种用于序列到序列模型预训练的去噪自动编码器。

作用：Default transform pipeline to apply to data. Can be specified in each corpus of data to override.还可以指定对特定数据的重写。

默认情况下什么都不用，即不对数据进行处理。

我不是很懂transform和pipeline的概念。
搜了一下，sklearn.pipeline.Pipeline、Language Processing Pipelines
在这个情景下，pipeline大概就是一些预处理步骤的组合。把原始数据放入pipeline中，就会自动执行那几个预处理步骤，来完成数据的预处理，再把结果输出pipeline？但是在普通的机器学习里，pipeline好像可以执行对参数的评估？

  • -save_data, --save_data
    要保存的对象的输出路径

  • -overwrite, --overwrite
    如果这个对象存在，是否需要重写。默认情况下不重写（False）。

  • -n_sample, --n_sample
    用多少个语料库/样本中多少个词来构建词汇表。-1表示用所有的，0表示不构建，默认情况是5000。

  • -dump_samples, --dump_samples
    在构建词汇表时转储样本。警告:这可能会减慢进程。默认情况下不这样做（即为False）。

  • -num_threads, --num_threads
    构建词汇表的并行线程数。默认是1。

  • -vocab_sample_queue_size, --vocab_sample_queue_size
    build_vocab转储路径中使用的队列大小。默认是100。
    转储文件是什么？愣着干嘛！不懂当然得学了

Vocab

  • -src_vocab, --src_vocab
    Path to save src (or shared) vocabulary file. Format: one or per line.

  • -tgt_vocab, --tgt_vocab
    Path to save tgt vocabulary file. Format: one or per line.

  • -share_vocab, --share_vocab
    Share source and target vocabulary.

Default: False

Transform/Filter

  • --src_seq_length, -src_seq_length
    Maximum source sequence length.

Default: 200

  • --tgt_seq_length, -tgt_seq_length
    Maximum target sequence length.

Default: 200

Transform/SwitchOut

Caution:This transform will not take effect when building vocabulary.

  • -switchout_temperature, --switchout_temperature
    Sampling temperature for SwitchOut. τ−1 in [WPDN18]. Smaller value makes data more diverse.

Default: 1.0

Transform/Token_Drop

  • -tokendrop_temperature, --tokendrop_temperature
    Sampling temperature for token deletion.

Default: 1.0

Transform/Token_Mask

  • -tokenmask_temperature, --tokenmask_temperature
    Sampling temperature for token masking.

Default: 1.0

Transform/Subword/Common

Attention:Common options shared by all subword transforms. Including options for indicate subword model path, Subword Regularization/BPE-Dropout, and Vocabulary Restriction.

  • -src_subword_model, --src_subword_model
    Path of subword model for src (or shared).

  • -tgt_subword_model, --tgt_subword_model
    Path of subword model for tgt.

  • -src_subword_nbest, --src_subword_nbest
    Number of candidates in subword regularization. Valid for unigram sampling, invalid for BPE-dropout. (source side)

Default: 1

  • -tgt_subword_nbest, --tgt_subword_nbest
    Number of candidates in subword regularization. Valid for unigram sampling, invalid for BPE-dropout. (target side)

Default: 1

  • -src_subword_alpha, --src_subword_alpha
    Smoothing parameter for sentencepiece unigram sampling, and dropout probability for BPE-dropout. (source side)

Default: 0

  • -tgt_subword_alpha, --tgt_subword_alpha
    Smoothing parameter for sentencepiece unigram sampling, and dropout probability for BPE-dropout. (target side)

Default: 0

  • -src_subword_vocab, --src_subword_vocab
    Path to the vocabulary file for src subword. Format: per line.

Default: “”

  • -tgt_subword_vocab, --tgt_subword_vocab
    Path to the vocabulary file for tgt subword. Format: per line.

Default: “”

  • -src_vocab_threshold, --src_vocab_threshold
    Only produce src subword in src_subword_vocab with frequency >= src_vocab_threshold.

Default: 0

  • -tgt_vocab_threshold, --tgt_vocab_threshold
    Only produce tgt subword in tgt_subword_vocab with frequency >= tgt_vocab_threshold.

Default: 0
