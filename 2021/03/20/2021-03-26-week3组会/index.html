Background and Problems

  • Traditional RNN can be used to seq to seq when the alignment between the inputs the outputs is known ahead of time.

  • But it has long term dependencies problem. (Gradient disappeared)

  • This also requires inputs and targets can be sensibly encoded with vectors of fixed dimensionality. But in many NLP tasks such as translation and question answering, the inputes and targets are always with different lengths.

a general model

encoder

decoder

《Sequence to Sequence Learning with Neural Networks》

  • To tackle long term dependencies, the author used LSTM.
  • Two LSTM. One for encoder, another for decoder.
  • The last hidden state of encoder is the first input of decoder.

reverse input

  1. Before reversing, the distances between every input word and target word are the same. Almost all of them need to span the entire sequence. After reversing, the average distance between each word and the target word has not changed. But the distance between the word at the front of the sequence and the target word is shortened which leads the model to generate a more precise result.

  2. From many papers, we can find that soft attention or alignment is very useful for seq2seq.

“RNN is a biased model, where later words are more dominant than earlier words.”《Recurrent Convolutional Neural Networks for Text Classification》

After reversing input, the first word in seq have the biggest influence in the hidden state. Then we use the hidden state to predict the first word which is kind of aligned. So we can get better performance.

shortages

  • all information of inputs is decompress to a fixed length vector whatever how long the input is which may leads to the loss of useful information.
  • only the first lstm unit uses hidden state of encoder

hidden state of decoder

  • $y{t-1}$ =$\mathbf{[\pmb e(y{t-1});\zeta(y_{t-1})]}^\top$

  • $\pmb e(y{t-1})$ ：Word embedding vector of $y{t-1}$

beam search

  • exhaustive search:Because of the large size of the vocab, we can’t calculate all the probable results and choose the best.
  • greedy search: At each step, choose the word with the highest probability of output. When we want a complete sentence, greedy search may lead to a suboptimal result.
