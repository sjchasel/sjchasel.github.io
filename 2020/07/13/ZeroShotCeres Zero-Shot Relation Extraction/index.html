Abstract

In many documents, such as semi-structured webpages, textual semantics are augmented with additional information conveyed using visual elements including layout, font size, and color.
在许多例如网页的这种半结构化文件中，文本的语义会附加一些可视化的信息，如字体大小、布局、颜色，来增强。
Prior work on information extraction from semi-structured websites has required learning an extraction model specific to a given template via either manually labeled or distantly supervised data from that template.
从半结构化的网页抽取信息的前期工作是通过手动标记的数据或从该模板远程监督的数据来学习特定于给定模板的提取模型。
In this work, we propose a solution for “zero-shot” open-domain relation extraction from webpages with a previously unseen template,including from websites with little overlap with existing sources of knowledge for distant supervision and websites in entirely new subject verticals.
在这项工作中，我们为从以前看不见的模板中的网页（包括与远程监管的现有知识资源很少重叠的网站）和全新垂直主题领域的网站中“零样本”开放域关系提取提出了一种解决方案。
Our model uses a graph neural network-based approach to build a rich representation of text fields on a webpage and the relationships between them, enabling generalization to new templates.
我们的模型用了一个基于图神经网络的方法去构建了网页上文本字段以及它们之间的关系的丰富表示，从而推广到新的模板。
Experiments show this approach provides a 31% F1 gain over a baseline for zero-shot extraction in a new subject vertical.
实验表明，这种方法在新的垂直对象中提供了在零样本提取基础上31％的F1增益。

1 Introduction

Semi-structured websites offer rich sources of high quality data across many areas of knowledge (Dong et al., 2014).
半结构化的网站提供了丰富的跨知识领域的高质量数据。
These websites present information via text that is accompanied by rich visual and layout features that can be generalized beyond a single website.
这些网站通过文本展现信息，并伴有丰富的可视化布局特征，这些功能可以推广到单个网站之外。（？）
However, most prior work on information extraction (IE) from websites has largely ignored most of these features, instead relying only on HTML features specific to an individual website (Ferrara et al., 2014).
然而，先前的大多信息抽取方面的工作都忽略了大多数功能（特征？），只是靠单个网站的html特性。
This requires training data for every website targeted for extraction, an approach that cannot scale up if training data must be manually created.
这就需要抽取每个网站的训练集数据，如果需要手动创建训练集的话，这种方法就无法大规模推广。

To circumvent manual data annotation, previous work used a distant supervision process requiring a knowledge base aligned to the website targeted for extraction (Gentile et al., 2015; Lockard et al.,2018), including for OpenIE extraction (Banko et al., 2007; Bronzi et al., 2013; Lockard et al., 2019).
为了避免手动进行数据的注释，之前的工作用了远程监督的过程，这需要知识库和要提取数据的目标网页对齐（？），包括OpenIE提取。
These methods, however, can only learn a website-specific model based on seed knowledge for the site, but cannot be generalized to the majority of websites with knowledge from new verticals, by long-tail specialists, and in different languages.
但是，这些方法只能基于站点的种子知识来学习特定于网站的模型，而不能用长尾专家和不同语言将新领域的知识推广到大多数网站。？？？

In this paper, we introduce the task of zero-shot relation extraction from semi-structured websites, in which a learned model is applied to extract from a website that was not represented in its training data (Figure 1).

本文介绍了从半结构化网页中进行零次关系抽取的任务，其中将一个学习模型应用于从一个未在其训练数据中展现（？）出来的网站中抽取信息。
Moreover, we introduce ZEROSHOTCERES, a graph neural network model that encodes semantic textual and visual patterns common across different training websites and can generalize to extract information from documents with never-before-seen templates and topics.
除此之外，我们介绍了ZEROSHOTCERES，一种图神经网络模型，可以对不同训练网站上常见的语义文本和视觉模式进行编码，并可以概括性地从具有从未见过的模板和主题的文档中提取信息。

Unlike unstructured text, which can be modeled as a sequence, or images, which can be modeled as a two-dimensional grid of pixels, it is not obvious how to operate over the many shapes and sizes of text fields on a semi-structured webpage.
与可以被建模成序列的非结构化文本或被建模未一个二维像素网格的图片不同，如何在半结构化的网页上操作具有多种形状或大小的文本区域并不明显。（明显？）
We illustrate our intuition using the webpage snippets in Figure 1: Despite their differences, each site uses alignment of relation and object strings, either vertically or horizontally, to help indicate relationships; in addition, relation strings are often more prominent than their objects, either in size or boldness.
我们使用图1中的网络片段来说明我们的直观感受：尽管他们有所不同，但每个网页的关系和字符串对象都是对齐的，以这种方式来说明他们的关系。除此之外，关系字符串也总是在字体和粗体方面比其他对象更突出。
Such features are semantically meaningful to readers and often consistent from site to site; thus, encoding them into the representation of webpages will allow us to generalize to unseen sites.
这些特征对于读者来说有着语义的意义，并且经常在站点之间保持一致；因此，将它们编码为网页的表示形式将使我们能够泛化到看不见的网站。

Our model, ZEROSHOTCERES, encodes these diverse feature types in a graph representation in which each text field becomes a node in a graph, connected by edges indicating layout relationships on the page.
我们的模型ZEROSHOTCERES把这些不同的特征类型进行编码，以一种图的形式表示，其中各个节点就是各个文本字段，边指示着页面上的布局关系。
This abstracts away the details of the page while maintaining the core visual structure presented to the reader.
这样可以抽象化页面的详细信息，同时将核心视觉结构保留下来呈现给读者。
A graph neural network is then applied to produce a new representation of each text field, informed by the surrounding page context.
随后，图神经网络被应用于为每个文本字段生产出一个新的展现形式，？？？
然后，通过周围的页面上下文，将图神经网络应用于每个文本字段的新表示。
This representation is then used to extract entities and relationships from the document. This allows us to extract not only in the closed-domain setting, but also allows us to conduct OpenIE on websites about entirely new subject verticals not seen during training.
然后，使用该表示法从文档中提取实体和关系。 这不仅使我们能够在封闭域设置中进行提取，而且使我们能够在全新的垂直领域上的网站上应用OpenIE进行训练。（？）

Our contributions are threefold: (a) We introduce a graph neural network model for webpage representation that integrates multi-modal information including visual, layout, and textual features, enabling generalization for IE from never-before seen websites.
我们的贡献有如下三点：（1）我们介绍了一种用于网页表示的图神经网络模型，它集成了包括视觉、布局和文本特征的多模式信息，实现了从没见过的网站中进行信息提取的通用化。
(b) We propose the first approach to enable Open Information Extraction from semi-structured websites without prior knowledge or training data in the subject vertical.
（2）首次提出了一种方法，可以从半结构化的网站中进行开放域知识抽取，而无需前期准备和垂直领域的训练数据。
(c) Our method works in both OpenIE and ClosedIE settings. We conduct evaluations showing the effectiveness of the technique and exploring the challenges of zero-shot semi-structured IE, achieving a 31% improvement in F1 compared to an OpenIE baseline. The graph model gives a 26% F1 boost when extracting according to a defined schema (ClosedIE).
（3）我们的方法在开放域知识抽取和封闭信息提取上都能发挥作用。我们进行的评估显示了我们方法的有效性，并探索了零样本半结构化的信息抽取任务。结果显示在OpenIE的基础上，F1提高了31%。当根据已经定义的方案进行信息抽取（封闭信息抽取）时，模型的F1提高26%

2 Related Work

DOM-based ClosedIE: The conventional approach to extraction from semi-structured websites is wrapper induction (Kushmerick et al.,1997), in which training data for documents from a given template is used to learn a rule-based extractor based on DOM (i.e., HTML) features to apply to other documents of the same template, extracting relations according to a pre-defined ontology (“ClosedIE”).
从半结构化网站中提取的常规方法是包装器归纳法（Kushmerick等，1997），其中使用来自给定模板的文档训练数据来学习基于DOM（即HTML）特征的基于规则的提取器 应用于相同模板的其他文档，并根据预定义的本体（“ ClosedIE”）提取关系。
Since this approach requires training data for each template targeted for extraction, recent work has focused on reducing the manual work needed per site.
由于此方法需要针对要提取的每个模板的训练数据，因此目前工作的重点是减少每个网站所需的手动工作。
Fonduer (Wu et al., 2018) provides an interface for easily creating training data, Vertex (Gulhane et al., 2011) uses semi-supervision to minimize the number of labels needed, LODIE (Gentile et al., 2015) and Ceres (Lockard et al., 2018) automatically generate training data based on distant supervision, and DIADEM (Furche et al., 2014) identifies matching rules for specific entity types.
Fonduer（Wu等人，2018）提供了一个轻松创建训练数据的界面，Vertex（Gulhane等人，2011）使用半监督来减少所需标签的数量，LODIE（Gentile等人，2015）和Ceres （Lockard et al。，2018）根据远程监督自动生成训练数据，而DIADEM（Furche et al。，2014）为特定实体类型识别匹配规则。

DOM-based OpenIE: WEIR (Bronzi et al., 2013) and OpenCeres (Lockard et al., 2019) offer OpenIE approaches to DOM extraction.

The latter method uses visual features in a semi-supervised learning setting to identify candidate pairs that are visually similar to known (relation, object) pairs;
后一种方法在半监督学习设置中使用视觉特征来识别在视觉上类似于已知对（关系，对象）的候选对。
however, the ultimate extraction model learned is still site specific and based on DOM features rather than the more generalizable visual or textual features.
但是，学习到的最终提取模型仍然是特定于站点的，并且基于DOM功能，而不是更通用的视觉或文字功能。
Pasupat and Liang (2014) present a zero-shot method for extraction from semi-structured webpages, but limit their work to extraction of entities rather than relationships and do not consider visual elements of the page.
Pasupat和Liang（2014）提出了一种从半结构化网页中提取内容的零样本方法，但其工作仅限于实体而非关系的提取，并且不考虑页面的视觉元素。

Multi-modal extraction: The incorporation of visual information into IE was proposed by Aumann et al. (2006), who attempted to learn a fitness function to calculate the visual similarity of a document to one in its training set to extract elements like headlines and authors.
多模态提取：Aumann等人提出将视觉信息合并到信息提取中。 （2006），他试图学习一种适应度函数，以计算文档与其训练集中的文档之间的视觉相似度，以提取标题和作者等元素。
Other recent approaches that attempt to address the layout structure of documents are CharGrid (Katti et al., 2018), which represents a document as a two-dimensional grid of characters, RiSER, an extraction technique targeted at templated emails (Kocayusufoglu et al., 2019), and that by Liu et al. (2018), which presents an RNN method for learning DOM-tree rules.
尝试解决文档布局结构的其他最新方法是CharGrid（Katti等人，2018），该方法将文档表示为二维字符网格，RiSER是一种针对模板化电子邮件的提取技术（Kocayusufoglu等人。 ，2019），以及Liu等人的研究。 （2018），提出了一种用于学习DOM树规则的RNN方法。
However, none of these address the OpenIE setting, which requires understanding the relationship between different text fields on the page.

The approaches most similar to ours are GraphIE (Qian et al., 2019) and the approach by Liu et al. (2019).

3 Problem and Approach Overview

3.1 Zero-shot relation extraction from semi-structured websites

We address the problem of extracting entities and the relationships between them as expressed by never-before-seen semi-structured websites.
我们解决了从从未见过的半结构化网站所表达的提取实体及其之间关系的问题。
A semi-structured website typically belongs to a subject vertical V , where V is a general field of knowledge such as movies, finance, or sports.
半结构化网站通常属于垂直主题V，其中V是电影，金融或体育等一般知识领域。
A semi-structured website consists of a set of detail pages sharing a similar template, each of which contains a set of facts about a page topic entity etopic.
半结构化网站由一组共享相似模板的详细页面组成，每个细节页面都包含有关页面主题实体etopic的一组事实。
The HTML document w defines a set of text fields T, which the web browser renders as a webpage according to the instructions defined in the HTML and any referenced auxiliary files such as CSS or Javascript.
HTML文档w定义了一组文本字段T，Web浏览器根据HTML中定义的指令以及任何引用的辅助文件（例如CSS或Javascript）将其显示为网页。
The text fields have both textual and visual features, described in Section 4.2.1.
文本字段具有文本和视觉功能，如第4.2.1节所述。

3.1.1 Relation Extraction关系抽取

Our goal is to extract (subject, relation, object) knowledge triples, where the subject is etopic, the object is a text field t ∈ T containing the name of an entity (or atomic attribute value), and the relation indicates the relationship between the two entities.
我们的目标是提取（主题，关系，对象）这样的知识三元组，主题就是e，对象是包含实体名称（或原子属性值（？？？））的文本字段t∈T，关系就是两个实体之间的关系
For this work, we assume the page topic entity has already been identified, (such as by the method proposed by Lockard et al. (2018) or by using the HTML title tag) and thus limit ourselves to identifying the objects and corresponding relations. We consider the following two settings:
对于这项工作，我们假设已经确定了页面主题实体（例如通过Lockard等人（2018）提出的方法或使用HTML标题标签），从而将我们自己限制在标识对象和对应关系上。 我们考虑以下两个设置：
Relation Extraction (ClosedIE): Let R define a closed set of relation types, including a special type indicating “No Relation”. Relation Extraction is the assignment of each text field t to one ri ∈ R, which indicates the relationship between the entity eobject mentioned in t and etopic.
关系提取（ClosedIE）：令R定义一组封闭的关系类型，包括指示“无关系”的特殊类型。 关系提取是将每个文本字段t分配给一个r i ∈R，它表示t中提到的实体e object 与e topic 。
Open Relation Extraction (OpenIE): Given a pair of text fields (i, j), Open Relation Extraction is a binary prediction of whether i is a relation string indicating a relationship between the entity eobject mentioned in j and etopic.
开放关系提取（OpenIE）：给定一对文本字段（i，j），开放关系提取是一种二进制预测，预测i是否为j中实体eobject和实体etopic之间的关系。

3.1.2 Zero-shot Extraction

Unlike prior work that requires the learning of a model specific to the semi-structured website targeted for extraction, we look at zero-shot extraction. Given a semi-structured website W targeted for extraction, zero-shot extraction is the learning of a model without any use of pages from W during training. We consider two zero-shot settings:
与先前的工作需要学习专门针对提取的半结构化网站的模型不同，我们将研究零样本提取。 给定目标为提取的半结构化网站W，零样本提取就是在训练过程中无需使用W的页面即可学习模型。 我们考虑两个零样本设置：
Unseen-Website Zero-shot Extraction is the learning of a model without any use of pages from W, but with pages from some other websit (s) from vertical V during training.
不可见网站的零样本提取是在训练过程中无需使用W中的页面，而是使用垂直V中的其他网站的页面来学习模型。
Unseen-Vertical Zero-shot Extraction is the learning of a model without any use of pages from W or of pages from any website with vertical V during training.
“不可见垂直领域的零样本提取”是在训练过程中无需使用W的页面或垂直V的任何网站的页面即可学习的模型。

3.2 Approach Overview


Figure 2 depicts our approach for zero-shot relation extraction (detailed in Section 5) leveraging a web page representation that will capture the similarities in visual and textual semantics across websites (Section 4). Our web page representation module first converts each page into a layout graph (Section 4.1) that abstracts away the details of the page structure while maintaining the adjacency relationships between text fields. We represent each text field with an initial feature vector of visual and textual attributes. This input is passed into a graph neural network that allows for information to flow between nodes, producing a new text field representation that captures contextual information (Section 4.2).

4 Web Page Encoder

The key idea behind our solution is to train webpage representations to capture the fundamental similarities in visual and textual semantics across websites to express relations, objects, and their relationships. The fundamental characteristics we capture, generalizable across templates and verticals, thus allow us to carry over our knowledge across websites and enable zero-shot extraction.
我们解决方案背后的关键思想是训练网页表示形式，以捕获整个网站在视觉和文本语义上的基本相似之处，以表达关系，对象及其关系。 我们捕获的基本特征可跨模板和垂直领域进行通用化，因此使我们能够跨网站继承我们的知识并实现零样本提取。
There are two key parts in our solution. First, we build a graph to capture the layout relationships in a more abstract form that allows us to more easily learn the common features across different sites such as the fact that relation strings are often to the left or above their objects (Section 4.1). Second, we apply a Graph Neural Network (GNN) to learn representations for each node capturing contextual information about its neighborhood on the webpage (Section 4.2), allowing information to flow through the nodes, providing context (e.g., flowing through “Cast” to a far-away node “Uma Thurman” via the closer node “Ethan Hawke” in Figure 3). This representation will be useful for relation extraction as described in Section 5.
我们的解决方案包含两个关键部分。 首先，我们建立一个图表以更抽象的形式捕获布局关系，这使我们能够更轻松地学习不同站点之间的共同特征，例如关系字符串通常位于其对象的左侧或上方（第4.1节）。 其次，我们应用图形神经网络（GNN）学习每个节点的表示形式，以捕获有关其在网页上邻域的上下文信息（第4.2节），允许信息流经节点，提供上下文（例如，流经“ Cast” 通过图3中更近的节点“ Ethan Hawke”将其移至较远的节点“ Uma Thurman”。 这种表示对于第5节中所述的关系提取将很有用。

6.1 Dataset

For both OpenIE and ClosedIE, our primary dataset is the extended version (Lockard et al., 2019) of the SWDE dataset (Hao et al., 2011), which contains gold labels for OpenIE extractions for 21 English language websites (each with one template) in three subject verticals (Movie, NBA, and University), with between 400 and 2,000 pages per site. We generated ClosedIE labels by converting the OpenIE labels to ClosedIE labels via manual alignment of OpenIE relations between websites, giving a set of 18 relations for the Movie vertical, 14 for NBA, and 13 for University. More information on training data creation and a complete listing of ClosedIE relations is available in the Appendix.
对于OpenIE和ClosedIE，我们的主要数据集是SWDE数据集（Hao等人，2011）的扩展版本（Lockard等，2019），其中包含21个英语网站的OpenIE提取的黄金标签。 （每个都有一个模板）三个主题（电影，NBA和大学），每个站点有400至2,000页。 我们通过手动对齐网站之间的OpenIE关系将Ope nIE标签转换为ClosedIE标签来生成ClosedIE标签，其中电影垂直关系为18，NBA为14，大学为13。 附录中提供了有关培训数据创建和ClosedIE关系的完整列表的更多信息。
We used three SWDE Movie sites (AMCTV, AllMovie, and IMDb) as a development set and did not evaluate on them for the reported results.
我们使用了三个SWDE电影网站（AMCTV，AllMovie和IMDb）作为开发集，并且未对报告的结果进行评估。

6.2 Experimental Settings

For each model tested (both our own and the baselines), we classify the training setting into the following categories indicating the level of vertical or site-specific knowledge used, in decreasing level of difficulty.
对于每个测试的模型（包括我们自己的模型和基线），我们将训练设置分为以下类别，这些类别指示所使用的垂直或特定于现场的知识的级别，从而降低了难度。

  • Level I–Unseen-Vertical Zero-shot (OpenIE only): A model is trained on sites from two of the three verticals (e.g. NBA and University) and applied to sites from the other vertical (Movie). This is the hardest case and is important when we wish to extract knowledge from new verticals where we do not have any prior knowledge or annotations.

  • Level II–Zero-shot with Vertical Knowledge:
    A model is trained on all sites but one (spanning Movie, NBA, and University) and then applied to the held-out site. As in cross-validation, experiments are repeated with each site having a turn being held out. It is easier than Level I but is still important for a new website that may not have data overlapping with other websites in the same vertical. For the ClosedIE setting, we train only on in-vertical sites.

  • Level III–Site-specific Knowledge: This is the traditional setting used by two of our baselines where we have seed knowledge overlapping with the website data to allow training a specific model for the website. Whereas Level I-II are both zero-shot settings, Level III is not, as it allows site-specific training data via weak supervision. (We do not present results using full supervision from manual annotations since it is known from prior work (e.g., Gulhane et al. (2011)) that full supervision from the target website yields highly accurate semi-structured extractors; we note that ZSCERES also achieves comparable results (∼ 0.95 F1) in this setting.

We repeated our experiments 10 times and we report the results averaged across the runs. For OpenIE, we follow the “lenient” scoring method for SWDE introduced by Lockard et al. (2019), scoring an extraction as correct if the relation string matches any of acceptable surface forms listed by the ground truth for that object.

Models are constructed in PyTorch (Paszke et al., 2017), with graph functions implemented in DGL (Wang et al., 2019) and optimization performed using Adam (Kingma and Ba, 2014) and a batch size of 20. For OpenIE, we use a hidden layer size of 25 for the GAT and 100 for the feed-forward layer. For ClosedIE, we use a hidden layer size of 200 for all layers. We use a 2-layer GAT and dropout of 0.25. We obtain visual features by rendering the page using the headless Chrome browser and querying the values using Selenium2.

Extraction Threshold:Since our zero-shot setting means we cannot use a development set of pages from the target site to tune the decision threshold, we instead set the threshold for each experiment to the value that attains the optimal F1 on the experiments where other sites were held-out.

OpenIE Postprocessing Rules: To ensure consistency among the extracted values, we keep only the highest confidence extraction in the case that the same text field is extracted as both a relation and object, or if multiple relations are extracted for the same object. In addition, some pages in the dataset contain relational tables, from which we sometimes extract the column headers as relations with the column contents as objects. While we believe a post-processing step could potentially recover these relational contents from our extractions, the SWDE data does not contain ground truth for such facts. Instead, we apply the heuristics described by (Cafarella et al , 2008) to identify these tables and remove them from our extractions.

8 Conclusion

We have introduced a zero-shot method for learning a model for relation extraction from semi-structured documents that generalizes beyond a single document template.
我们介绍了一种零样本方法，用于学习从半结构化文档中提取关系的模型，该模型可以推广到单个文档模板之外。
Moreover, this approach enables OpenIE extraction from entirely new subject verticals where no prior knowledge is available.
此外，这种方法可以从没有可用先验知识的全新垂直行业中应用OpenIE进行提取。
By representing a webpage as a graph defined by layout relationship between text fields, with text fields associated with both visual and textual features, we attain a 31% improvement over the baseline for new-vertical OpenIE extraction.
通过将网页表示为由文本字段之间的布局关系定义的图形，同时将文本字段与视觉和文本功能相关联，我们将新垂直领域中的OpenIE提取的基线提高了31％。
Future extensions of this work involve a more general pre-training objective allowing for the learned representations to be useful in many tasks as well as distantly or semi-supervised approaches to benefit from more data.
这项工作的未来扩展涉及更一般的预训练目标，使学习的表示形式在许多任务中有用，以及从更多数据中受益的远程或半监督方法
