Automatic Keyphrase Extraction via Topic Decomposition
2020-07-20|

Abstract

传统的关键词抽取是把文档转换成图之后，在图上进行一次随机游走，对单词打分。一个事实是，文本或文档可以被分解为多个语义主题内容，因此作者想到可以在不同主题上进行随机游走。先对主题打分，（在给定文档主题分布的情况下）再对主题中的单词打分。

Introduction

提取关键词的方法：有监督和无监督
有监督的方法：分类问题。人工先标记有哪些关键词，再对样本进行分类。但人工标注太麻烦了
无监督：根据文档构建词图，再根据随机游走的方法对词进行排序（如PageRank）
作者发现同一个词语在不同的语义主题下可能有不同的含义，如果不考虑主题直接对所有单词进行排序的话，会有两个问题：

  1. 挑选出来的单词虽然与其他单词都关系密切，但可能并不符合文章的主题
  2. 挑选出来的单词可能符合几个主题中的一个，但不符合文章的实质主题
    因此他针对不同主题来进行排序。先在不同主题下得到单词的重要性排名，再提取文档的关键词。Topical PageRank有以下两个步骤：
  3. 建立一个主题解释器来获取单词和文档的主题。
  4. 执行TPR以提取文档的关键短语。

当滑动一个W宽度的窗口时，我们在每个位置添加从第一个单词指向窗口内其他单词的链接。由于关键词通常是名词短语，所以我们只在词图中添加形容词和名词。

实验

数据集

  • NEWS：包含了308篇DUC2001新闻文章（Over等人，2001），其中有2488篇手动注释的关键短语。每个文档最多有10个关键短语。
  • RESEARCH：包含2000篇研究文章摘要和19254个手动标注的关键词。

作者又用了维基百科上的一大堆文章来构建LDA模型（从大规模文档中推断文章的主题）

评价指标

标准和提取的关键短语中的单词都被简化为基本形式
英文词干提取算法(Porter stemmer)： 主页的一句话：Porter stemmer 并不是要把单词变为规范的那种原来的样子，它只是把很多基于这个单词的变种变为某一种形式！换句话说，它不能保证还原到单词的原本，也就是”created”不一定能还原到”create”，但却可以使”create” 和 “created” ，都得到”creat” ！

  1. 精度/召回率/F-度量
    p=ccorrect/cextract
    r=ccorrect/cstandard
    f=2pr/(p+r)
符号 含义
ccorrect 通过一种方法提取的正确关键短语的总数
cextract 自动提取的关键短语的总数
cstandard 类标记的标准关键短语的总数

由于排序的指标决定着最后的结果，也即影响着算法的性能。这种指标没有考虑提取的关键短语的顺序，所以作者又构建了两个额外的指标

  1. binary preference measure (Bpref)二进制优先度量
    对于一个文档，如果通过一种方法在M个提取的关键字短语中有R个正确的关键字短语，其中R是正确的关键字短语，n是不正确的关键字短语，则Bpref的定义如下
    M：提取的关键字短语数量
    R：正确的关键字短语数量
    r：正确的关键字短语
    n：不正确的关键字短语

  2. 平均互易秩（MRR）
    用于评估每个文档的第一个正确关键字是如何排名的

    d：文档
    D：用于关键字提取的文档集

结论

  1. 在本文中，我们利用维基百科的LDA获取潜在话题。我们设计了使用其他机器学习方法和其他知识库获取主题，并研究了对关键短语提取性能的影响。
  2. 本文将主题信息集成到PageRank中。我们计划在基于疗法的绘图算法中考虑主题信息，如HITS（Kleinberg，1999）。
  3. 在本文中，我们使用维基百科来训练LDA，假设维基百科是人类知识的一个广泛的快照，可以涵盖新闻和研究中谈论的大多数话题。事实上，学习主题对学习语料库的依赖性很强。我们将探讨语料库选择对用TPR训练LDA进行关键短语提取的影响。
