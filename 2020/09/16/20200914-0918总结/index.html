Semi-Supervised Learning for Neural Keyphrase Generation

Summary

While seq2seq model demonstrate good performance on keyphrase generation, it heavily relies on massive amounts of labeled data for model training. But for a new domain, the labeled data is not enough for model training. To solve this problem, author propose semi-supervised keyphrase generation methods based on seq2seq model.

  1. unlabeled documents are first tagged with synthetic keyphrases obtained from unsupervised keyphrase extraction methods or a self-learning algorithm, and then combined with labeled samples for training.
  2. a multi-task learning framework to jointly learn to generate keyphrases as well as the titles of the articles.

In addition to overcoming the problem of large number of labeled data needed to train models, it also alleviates the overfitting problem caused by using small-scale labeled data to train models. It performs well on existing data sets.

Research Objective

To solve the problem that the keyword generation model needs too much annotation data to train

Background and Problems

Keywords used to extract documents have applications in a variety of fields. In the past, the extraction of keyphrase was to specify a set of keywords in advance, and then extract the keywords that can summarize the article. This method can only generate the vocabulary that exists in the article. Keyphrase generation proposed by Meng can generate words that are not in the text. But the seq2seq model he used requires a lot of labeled data to train model. There’s not that much labeled data for a new domain. For this reason, the author studies the solution…

Amount of Unlabeled Data

When increasing the amount of unlabeled data, model performance is further improved. This is because additional unlabeled data can provide with more evidence on linguistic or context features and thus make the model, especially the encoder, have better generalizability.

One Size Does Not Fit All: Generating and Evaluating Variable Number of Keyphrases

Summary

seq2seq model are often used in keyphrase generation. While seq2seq model can not generate multiple keyphrases, only can generate one sequence at a time, we usually use beam search to generate many keyphrases and take the first few. However, this method cannot dynamically change the number of output keyphrases depending on the length of the text, and it often outputs keyphrases with similar meaning.

So author propose a recurrent generative model which can generate different number of keyphrases according to different texts, and the generated keyphrases are more diverse.A new method to evaluate the model is also proposed. The authors’ model performs better on both old and new metrics than on strong baselines.

Research Objective

For flexibility in generating keyphrases based on document’s length and reasonable evaluation of the model.

Background and Problems

exsiting method for keyphrase generation

  1. The number of keyphrases cannot automatically change depending on the length of the text
  2. Generate a large number of keyphrases at one time and then take the first few, which will lead to the repetition of the meaning of keyphrases and lack of diversity
  3. When evaluating the production of keyphrases in an article, the same evaluation criteria should not be used to evaluate it.
    For example, a very short article with five keyphrases is enough, but a very long article with only the first five key phrases that match the content of the article will not produce good results

Evaluation


The previous evaluation measure, k, is a constant between yk and y, which is less flexible.
The authors define two new indicators:

  • F1@O：O=the true number of keyphrases
  • F1@M：M=the number of predict keyphrases
