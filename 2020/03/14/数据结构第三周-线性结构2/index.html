单链表及其实现

定义

一个结点（Node）包括两部分：数据域（data）和地址域（引用域、链域、next）。数据域存储数据元素，地址域存储它的后继的地址信息。最后一个元素的后继为空。除了第一个结点其他每个结点的对象引用都存储在前一个结点的next中。
加上一个没有意义的头结点，为了方便查找。

双向链表与循环链表

双链表

定义

每个结点有两个地址域，分别指向前驱结点和后继结点

双链表结点类：

插入

  1. q.prev = p.prev;
    a~i-1~成为q的前驱

  2. p.prev.next = q;
    q成为a~i-1~的后继

  3. p.prev = q;

  4. q.next = p;

删除

  1. p.prev.next = p.next;
  2. p.next.prev = p.prev;

循环链表

循环单链表

将单链表中最后一个结点的next域指向head，形成环形结构。

  1. 空循环链表：head.next = head;
  2. 循环单链表：rear.next = head;

循环单链表与单链表的不同

  1. 单链表p指向尾结点时：p.next = null;
    循环单链表p指向尾结点时：p.next = head;
  2. 循环单链表设置尾指针可以更方便地找开始结点和终端结点：
    开始结点：rear.next.next
    终端结点：rear

顺序表与链表的比较

时间性能比较

随机访问操作

顺序表：O（1）
链表：O（n）

插入和删除操作

链表：插入或删除后继结点O（1）
顺序表：O（n）

一般规律

  • 若线性表需要频繁进行查找却很少进行插入或删除操作，或者操作和数据元素的位序密切相关时，宜采用顺序表作为存储结构。
  • 若线性表需要比较频繁的插入和删除操作，比较适合采用链表。

空间性能比较

结点存储密度

顺序表：只需存储数据元素引用，存储空间利用率高。
链表：除了存储数据元素的引用变量外，还要存储元素之间逻辑关系的引用变量，引用的结构性开销占了整个结点的一半。

存储空间分配

顺序表：需要分配合适长度的存储空间
链表：不需要进行固定长度的分配，对元素个数无限制

一般规律

  • 当线性表中元素个数变化较大或未知，用链表
  • 若事先知道线性表的大概长度，用顺序表
