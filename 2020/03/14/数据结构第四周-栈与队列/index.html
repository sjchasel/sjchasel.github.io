2020-03-14|

栈的概念及应用

  • 队列只能在一端进行插入，另一端删除，在线性表的两端进行，即先进先出

栈的定义

  • 一种特殊的线性表。栈只能在栈顶操作，插入和删除操作只允许在线性表的一端进行，即先进后出。允许操作的一端是栈顶，不允许的是栈底。表中没有元素的时候是空栈。
  • 假设S=(a0,a1,a2,a3,…,an-1)。则a0是栈底元素，an-1是栈顶元素。标识栈顶位置的指针叫栈顶指针。栈中元素按a0，a1,…,an-1的顺序进栈，退栈的第一个元素应为栈顶元素an-1。栈也叫后进先出表。

栈的操作

  • 删除也叫出栈/弹出。插入也叫进栈/入栈/压入。
  • 在空栈中最先插入的元素总是放在栈的底部，当所有元素被弹出后它才被删除。
  • 不管什么操作都是栈顶指针在移动。

栈的溢出

  • 栈满时进栈为“上溢”，出错状态
  • 栈空时退栈为“下溢”，可能是正常现象，下溢用来作为程序控制转移的条件。

栈的分类

顺序栈

定义

  • 栈的顺序存储结构叫顺序栈，是运算受限的线性表，因此可以用数组实现顺序栈。
  • 栈底位置固定不变，所以可以将其设置在数组两端的任何一个端点。一般用一个整型变量top表示栈顶指针。
  • 空栈时栈顶指针为-1，元素入栈时，先将栈顶指针+1，再把元素放入栈顶指针指向的 位置。
  • 出栈时，先取出元素的值，再把栈顶指针的值-1。

顺序栈与链式栈的比较

实现链式栈和顺序栈的操作的时间复杂度都是O（1）

  • 初始时，顺序栈必须说明一个固定长度。当栈不满时，造成了空间的浪费。
    链式栈的长度可变，不需要预先设置，相对比较节省空间。但是每个结点中有一个指针域，产生了结构开销。
  • 当需要多个栈共享时，顺序存储中可以充分利用顺序栈的单向延伸性。用一个数组存两个栈，每个栈从各自的端点向中间延伸，减少了浪费的空间。但这只有当两个栈的空间需求相反时才有效。

队列的概念与应用

定义

  • 队列也是操作受限的线性表。只允许在一端插入（这端叫队头front），在另一端删除（这端叫队尾rear）。
  • 在队列中加入元素a0，a1，…,an-1后，a0是队头元素，an-1是队尾元素。所以退出队列的次序只能是a0，a1，…,an-1.即先进先出。
  • 队列也叫先进先出的线性表，简称FIFO表。

操作

  • 入队叫队列的插入
  • 出队叫队列的删除
  1. 开始时，队列为空，front队头指针与rear队尾指针都指向0位置。
  2. a入队时，a放在0位置，rear向后移到1位置；b入队时，b放在1位置，rear向后移到2位置;…
  3. a出队时，0位置空了，front指针移到1位置指向b
  4. b和c都出队后，front和rear都指向3位置

数据元素

队列的数据元素是n个相同类型的数据元素组成的有限序列。

顺序队列

定义

  • 队列的顺序存储结构就叫顺序队列
  • 运算受限的顺序表。用一个数组空间存放当前队列中的元素
  • front指针指向队头元素，rear指针指向队尾元素在数组空间中的位置。初值可以是0或-1

操作

  • 入队时：rear+1，新元素插入rear所指的位置
  • 出队时：删去front指的元素，front+1，返回被删元素
循环队列
假溢出的定义



头尾指针不断增加而不减小或只减小却不增加，使被删除元素的空间无法被重复利用。

解决假溢出的办法

使用循环向量，存储在循环项链里的队列被称为循环队列。将队列设计成逻辑上首尾相接的循环队列，就可以循环使用顺序队列的连续存储单元。

操作

入队：rear=（rear+1）%maxsize
出队：front=（front+1）%maxsize
初始化：front = rear = 0；
队列空：front == rear；
队列满：(rear + 1) % MaxSize == front；
元素数：num = maxsize-1
