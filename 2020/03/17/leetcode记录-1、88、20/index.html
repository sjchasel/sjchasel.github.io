题目

给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那 两个 整数，并返回他们的数组下标。
你可以假设每种输入只会对应一个答案。但是，你不能重复利用这个数组中同样的元素。

示例:
给定 nums = [2, 7, 11, 15], target = 9
因为 nums[0] + nums[1] = 2 + 7 = 9
所以返回 [0, 1]

解法1：双重循环暴力解法

  • 时间复杂度为O（n^2^）
  • 有返回的函数第一步写一个return null;
  • 注意在第二层循环中，j从i+1开始，不需要从0开始再去掉i=j相等的情况（这样做过于复杂）。从i+1开始，i，j就可组合出所有情况，因为它们是对同一个数组操作。
  • 注意容错处理，先判断nums的长度。

哈希表

基本概念

这道题好像好的解法都需要用哈希表，于是去补充一点知识。

  • Map<K, V>
    是一个接口，以键值（Key）-数值（Value）对应形式存储数据。与数组通过下标对应内容索引不同，在Map中通过对象（key）对对象（value）进行索引
  • HashMap
    是Map<K, V>的实现类。它存储数据用的是哈希表结构，键值唯一不重复。
  • 哈希函数
    建立key与int值映射关系的函数。它把任意的对象映射成一个int数值供哈希表用。
  • 哈希表
    一个数组。当放入元素时，要先通过哈希函数获得一个int值，处理后作为它的存放位置。
  • 哈希碰撞
    int值有限，当不同对象获得同一个int值时就是发生了哈希碰撞。

常用方法

Type Method
Value get(Object key) 返回键映射的值，如果该键没有映射值，则返回null
Value put(K key, V value) 将键和值建立映射关系,如果键是第一次存储，就直接存储元素，返回null; 如果键不是第一次存在，就用值把以前的值替换掉，返回以前的值
Value remove(Object key) 如果对应键存在映射关系的值，则将其移除，并返回值
boolean containsKey(Object key) 是否存在特定的key
boolean containsValue(Object value) 是否存在特定的value
boolean isEmpty() 判断集合是否为空
int size() 返回集合中的键值对的对数

解法2：两遍哈希表

  • 保持数组中的每个元素与其索引相互对应的最好方法是哈希表。
  • 哈希表可以以空间换速度
  • throw new IllegalArgumentException(“No two sum solution”);抛出这个异常说明方法传入一个非法的或者不合适的参数。
    举个例子：getUser(int username)方法，不允许传入空字符串或者null。但是有个调用的方法，没做检查，传入了null或者空字符串，这时候getUser方法就应该要抛出IllegalArgumentException告诉调用者：hi！这个参数不能为empty或者null。
    java.lang.IllegalArgumentException继承至RuntimeException，所以它是一个unchecked异常，它不需要在方法里加throws声明！
  • 如果换成return null;，运行速度下降
  • 时间复杂度O（n）

题目

给定一个只包括 ‘(‘，’)’，’{‘，’}’，’[‘，’]’ 的字符串，判断字符串是否有效。

有效字符串需满足：

左括号必须用相同类型的右括号闭合。
左括号必须以正确的顺序闭合。
注意空字符串可被认为是有效字符串。

我的在鳝的帮助下好不容易挤出来的垃圾解法

  • 基本的想法是遇到左括号压入栈，遇到右括号与栈顶的左括号比较，若匹配弹出，最后看栈里面还有没有剩余的括号。
  • 由于要判断是左括号还是右括号，以及判断是否匹配，所有有些了几个方法。

  • 虚拟值#解决了当输入一个单独的右括号的情况
