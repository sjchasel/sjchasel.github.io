URI与URL

URI是统一资源标识符，URL是统一资源定位符。URI包括URL与URN（统计资源名称，只命名资源，但不定位）。
http://github.com/favicon.ico 既是URI也是URL，它包括访问协议、访问路径（根目录）和资源名称。

超文本

网页的源代码HTML就可以称作超文本。

协议类型

包括http、https、ftp、smb…

HTTP：超文本传输协议

HTTPS：HTTP的安全版，经过它传输的内容都经过了SSL加密。是大势所趋。

  • 建立安全通道
  • 确认网站的真实性

HTTP请求过程

输入URL可看见页面内容，这个过程是向网站所在的服务器发送了一个请求（Request），服务器接收到请求后，对其进行处理和解析，再返回对应的响应（Response），传回给浏览器。

请求

  • 请求方法
    • GET
      比如在百度中搜索就是一个GET请求。URL中包含了请求的参数信息。提交的数据有长度限制（1024字节）。
    • POST
      一般在提交表单时发起。数据以表单的形式传输，而不提现在URL中
    • 等
  • 请求的网址
    即URL
  • 请求头
    • Accept：请求报头域，指定客户端可接收哪些类型的信息
    • Host：发送请求的原始服务器或网关的位置
    • Cookies：主要功能是维持当前访问会话
    • Referer：标识这个请求从哪个页面发来的
    • User-Agent：爬虫加上这个信息可以伪装成浏览器，而不会被识别为爬虫
    • Content-Type：提交数据的方式
  • 请求体

响应

可以根据状态码判断访问状态
常用的响应头信息：

  • Date
  • Last-Modified
  • Server
  • Expires

响应体中就是需要的内容
